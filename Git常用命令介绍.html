在当前目录初始化Git仓库 [bash]git init[/bash] 跟踪文件，添加文件到暂存区 [bash]git add[/bash] 提交暂存区文件到仓库 [bash]git commit[/bash] 常用参数：
-a:提交工作区已经修改的文件和暂存区的文件到仓库
--amend:修改最后一次提交，或者说将上次提交后的修改追加到上次提交
克隆仓库到本地 [bash]git clone git@github.com:YOUR_USER_NAME/REPO_NAME.git[/bash] 或者
[bash]git clone https://github.com/YOUR_USER_NAME/REPO_NAME.git[/bash] 查看状态 [bash]git status[/bash] 查看提交历史 [bash]git log[/bash] 文件比较，commit缺省的情况下和HEAD比较 [bash]git diff [--cached/staged] [][/bash] 常用参数:
--cached:比较暂存文件和已经提交文件
--staged:比较暂存文件和已经提交文件
移除文件 [bash]git rm[/bash] 从暂存区及仓库中移除文件，并且以后不再跟踪 [bash]git rm --cached file[/bash] 移动或重命名 [bash]git mv src dest[/bash] 取消暂存 [bash]git reset HEAD file[/bash] 重置工作目录到上次提交的状态 [bash]git reset --hard HEAD[/bash] 恢复单个文件到上次提交的状态 [bash]git checkout -- file[/bash] 回退到上一次提交，如果马上commit，相当于拷贝一个旧的版本 [bash]git revert HEAD[/bash] 创建分支 [bash]git branch branch_name[/bash] 参数说明：
-b:创建分支并切换到新的分支，相当于 [bash]git branch branch_name git checkout branch_name[/bash] 查看本地分支 [bash]git branch[/bash] 参数说明：
-r:查看远程分支 切换分支 [bash]git checkout branch_name[/bash] 合并分支branch_name到当前分支 [bash]git merge branch_name[/bash] 查看当前分支和branch_name分支的差异 [bash]git diff branch_name[/bash] 删除分支branch_name [bash]git branch -d branch_name[/bash] 查看远程仓库 [bash]git remote git remote show[/bash] 查看远程仓促repo_name细节 [bash]git remote show repo_name[/bash] 添加远程仓库到本地 [bash]git remote add local_name(origin) remote_url[/bash] 修改远程仓库在本地的名称 [bash]git remote rename old_name new_name[/bash] 删除远程仓库在本地的连接 [bash]git remote rm local_name(origin) 抓取远程仓库中更新的数据 [bash]git fetch local_name(origin)[/bash] 抓取远程仓库分支到本地并且合并 [bash]git pull local_name(origin)[/bash] 相当于 [bash]git fetch local_name(origin) git merge local_name(origin)/branch_name[/bash] 推送到远程仓库 [bash]git push local_name(origin) local_branch_name:remote_branch_name[/bash] 显示标签 [bash]git tag[/bash] 参数说明：
-a:增加标签 -d：删除标签 -l：显示标签列表 -m:增加标签时的备注

日常使用例子

个人仓库

  1. 在Github新建一个仓库REPO_NAME
  2. 在本地目录初始化仓库：[bash]git init[/bash]
  3. 将本地仓库和远程仓库建立连接：[bash]git remote add origin git@github.com:YOUR_USER_NAME/REPO_NAME.git[/bash]
  4. 增加/修改/删除文件
  5. 添加修改到暂存区：[bash]git add[/bash]
  6. 提交到本地仓库:[bash]git commit -m[/bash]
  7. 推送到远程仓库:[bash]git push origin master[/bash]

fork 他人仓库

  1. 在源仓库的Github页面点击fork，fork到个人仓库
  2. 克隆到本地：[bash]git clone git@github.com:YOUR_USER_NAME/REPO_NAME.git[/bash]

fork的仓库和源仓库同步

  1. 增加本地仓库到源仓库的远程连接:[bash]git remote add upstream git@github.com:ORIGIN_OWNER/REPO_NAME.git[/bash]
  2. 将远程仓库同步到本地：[bash]git fetch upstream[/bash]
  3. 将同步到本地的远程仓库合并到本地master分支：[bash]git checkout master git merge upstream/master[/bash]
  4. 将同步推送到Github：[bash]git push origin master[/bash]
